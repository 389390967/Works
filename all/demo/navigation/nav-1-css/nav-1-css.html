  • transition 过渡
  • Internet Explorer 9 以及更早版本的浏览器不支持 transition 属性。
  • 默认值：all 0 ease 0
  • transtion-property 当:hover发生时要过渡的CSS属性。
  • transtion-duration 过渡效果需要花费的时间（以秒或毫秒计）。
  • transtion-timing-function 过渡效果的速度曲线。
  • transtion-delay 设置过渡效果延迟多久开始（以秒或毫秒计）。
  • transform 改变
  • Internet Explorer 9 支持替代的 -ms-transform 属性（仅适用于 2D 转换）。
  • Safari 和 Chrome 支持替代的 -webkit-transform 属性（3D 和 2D 转换）。
  • Opera 只支持 2D 转换。
  • transform-scaleY() scale:比例
  • 通过设置 Y 轴的值来改变元素高度的比例。
  • 例如：元素写了这个样式后{height:100px;transform-scaleY(2)}，高度将变成了200px。
  • transform-translateZ()
  • translate的本义是转化、翻译，这让我很难理解。本例子transform-translateZ(0)的作用等同于z-index(0)。
  • transform-origin 改变的起点
  • 该属性必须与 transform 属性一同使用。
  • 默认值：50% 50% 0
  • transform-origin: x-axis y-axis z-axis（axis：轴）。
  • 本导航栏的:hover效果是设置了transform-origin: 50% 0; 即transform-scaleY(0)变到transform-scaleY(1)的起点是元素的顶部，元素的高度比例由顶部改变至底部。
  • 如果是设置了transform-origin: 50% 100%; 即transform-scaleY(0)变到transform-scaleY(1)的起点是元素的底部，元素的高度比例由底部改变至顶部。
  • 如果是设置了transform-origin: 50% 50%; 即transform-scaleY(0)变到transform-scaleY(1)的起点是元素的中部，元素的高度比例由中部改变至顶部和底部。

    起始点固定值为A(0,0),
    终止点固定值为D(1,1),
    剩下的中间两点B(x1,y1),C(x2,y2)
    也就是所要动态操控的两个点了,对应cubic-bezier (x1,y1,x2,y2)中的四个参数,
    x1值越小和y1值越大起始速速就越快
    x2值越小和y2值越大结束速速就越慢
    通过改变B,C两点的坐标值来动态生成一条贝塞尔曲线表示动画中的速度变化。
    linear 对应自定义cubic-bezier(0,0,1,1)，效果为匀速直线（即B=A,C=D）；
    ease 对应自定义cubic-bezier(.25,.1,.25,1),效果为先慢后快再慢；
    ease-in 对应自定义cubic-bezier(.42,0,1,1),效果为先慢后快；
    ease-out 对应自定义cubic-bezier(0,0,.58,1),效果为先快后慢；
    ease-in-out 对应自定义cubic-bezier(.42,0,.58,1),效果为先慢后快再慢。
